# Заметка: `23288-j50.pdf`

- **Файл:** `docs/specs/23288-j50.pdf`
- **Спецификация:** `TS 23.288`
- **Приоритет:** `P3`
- **Кому читать:** `Студент 2`
- **Неделя:** `этап 2`

## Кратко о документе
архитектурные расширения 5GS для сетевой аналитики.

## Связь с 3GPP NMS
объясняет, как аналитика встраивается в архитектуру управления.

## Чем полезен на практике
помогает спланировать этап после MVP (MDA и продвинутая аналитика).

## Для какого функционала
аналитика, прогнозирование, data-driven оптимизация.

## Что изучить в документе
- роль аналитических сервисов в архитектуре 5GS;
- какие данные нужны для аналитики;
- границы между сбором данных и аналитикой.

## Как применить в проекте
- планировать этап 2 с учетом MDA;
- не смешивать базовый мониторинг MVP и сложную аналитику.

## Что подготовить после чтения
- черновой план внедрения аналитики после MVP;
- список данных, которые нужно начать сохранять заранее.
