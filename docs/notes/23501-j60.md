# Заметка: `23501-j60.pdf`

- **Файл:** `docs/specs/23501-j60.pdf`
- **Спецификация:** `TS 23.501`
- **Приоритет:** `P3`
- **Кому читать:** `Студент 1`
- **Неделя:** `этап 2`

## Кратко о документе
системная архитектура 5G (Stage 2).

## Связь с 3GPP NMS
дает контекст, какие сетевые функции и интерфейсы должны быть видны для управления.

## Чем полезен на практике
помогает корректно привязать объекты управления к реальной архитектуре сети.

## Для какого функционала
inventory, topology, междоменная интеграция (RAN/Core).

## Что изучить в документе
- базовые функциональные блоки 5GS;
- ключевые интерфейсы между блоками;
- какие элементы важны для управления и видимости в NMS.

## Как применить в проекте
- использовать для корректной топологии и связей;
- сверять, что модель inventory не противоречит архитектуре 5GS.

## Что подготовить после чтения
- схема "сетевой блок -> объект управления";
- карта критичных интерфейсов для мониторинга.
