# Неделя 06. Безопасное изменение конфигурации

## Цель недели
Сделать одно надежное изменение конфигурации от API до узла.

## Задачи

Студент 1:
- реализует шаги `lock/edit-config/validate/commit/unlock`;
- добавляет откат при ошибке.

Студент 2:
- делает сущность запроса на изменение;
- добавляет журнал кто и что изменил.

## Что читаем
- `docs/specs/28510-j00.pdf`
- `docs/specs/28532-j30/TS28532_ProvMnS.yaml`

## Результат недели
- рабочий сценарий изменения параметра на узле.

## Проверка
При ошибке изменения узел не остается в промежуточном состоянии.
