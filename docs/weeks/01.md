# Неделя 01. Границы проекта

## Цель недели
Определить, что именно входит в минимальный MVP, чтобы не перегрузить проект.

## Задачи

Студент 1:
- готовит список узлов для пилота;
- определяет минимальный набор `NETCONF` операций;
- фиксирует риски подключения.

Студент 2:
- фиксирует пользовательские сценарии;
- задает минимальные требования к API и журналам;
- описывает границы функций MVP.

## Что читаем
- `docs/specs/28500-j00.pdf`
- `docs/specs/28530-j00.pdf`
- `docs/specs/28533-j40.pdf`

## Результат недели
- утвержденный список функций MVP;
- список того, что переносим на этап 2.

## Проверка
Команда одинаково понимает, что считается готовым MVP.
