# Неделя 07. Простая проверка схемы

## Цель недели
Отсеивать некорректные запросы до отправки в сеть.

## Задачи

Студент 1:
- проверяет, что нужный модуль поддерживается узлом;
- проверяет путь к изменяемому параметру.

Студент 2:
- проверяет формат данных запроса;
- возвращает понятные ошибки в API.

## Что читаем
- `docs/specs/28623-j60/_3gpp-common-top.yang`
- `docs/specs/28623-j60/_3gpp-common-yang-types.yang`
- `docs/specs/28659-j00.pdf`

## Результат недели
- минимальная валидация перед CM.

## Проверка
Некорректный запрос блокируется до вызова `NETCONF`.
